# 基于RISC-V指令集架构的CPU

## 项目架构

- 核心
  - ALU：算术逻辑单元。根据算术逻辑操作类型进行对应的运算。
  - CU：控制单元。解码指令，并根据指令执行内存读写、计算等操作。
- 内存（Block RAM）
  - data RAM
  - instruction RAM
- 外设接口
  - LED接口
  - UART接口
  - VGA接口
  - Keyboard接口
- 流水线（五级）
  - Instruction Fetch（IF）
  - Instruction Decoding（ID）
  - Execution（EX）
  - Memory Access（MEM）
  - Write Back（WB）
  - 寄存器文件
- 总线
  - 总线仲裁
  - 总线解码
  - 总线多路选择
  - DMA
- CACHE
  - 数据Cache
  - 指令Cache
- 优化内容
  - 分支预测器
    - RAS
    - 锦标赛预测器
    - Gshare算法
    - 局部预测
  - 内存保护
    - 数据页控制（MMP）
    - 物理内存保护（PMP）
    - 控制状态寄存器（CSR REG）
  
## 项目数据流

### 核心流动

#### 1.（Core）指令-数据流

- 指令流入CU，CU读取操作码（OPCODE）并且进入对应的状态解码指令，从而将相应的ALU操作状态置位；
- ALU检测到状态变化，读取操作数，根据操作类型执行运算，将运算结果存入结果寄存器

#### 2.流水线

- 取指：根据程序计数器取出指令和下一条指令的位置，读取内存数据，读取采取的分支，读取分支目标
- 译码：该阶段下，控制单元接收指令并译码，将指令内容放入寄存器等待执行
- 执行：该阶段下，ALU执行计算、跳转指令，更新寄存器
- 访存：该阶段下，CPU根据寄存器中的数据计算写回地址，执行数据冒险等
- 写回：该阶段下，CPU根据写入地址将数据写入内存或者寄存器，一条指令结束
